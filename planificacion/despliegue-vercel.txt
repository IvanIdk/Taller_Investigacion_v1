PRE-REQUISITOS
--------------
  - Cuenta en https://vercel.com
  - Repositorio en GitHub/GitLab/Bitbucket (recomendado)
  - Build local exitoso: cd frontend && npm run build  (verificado OK)

CONFIGURACIÓN DEL PROYECTO (OBLIGATORIO)
----------------------------------------
  1. Ir a https://vercel.com/new
  2. Importar el repositorio Ivanldk/Taller_Investigacion_v1
  3. *** Root Directory = frontend ***  (Settings → General si ya existe el proyecto)
  4. Framework Preset: Next.js (detectado automáticamente)
  5. Build Command: npm run build
  6. Install Command: npm ci
  7. NO usar vercel.json en la raíz del repo (solo frontend/vercel.json)

  Si Root Directory queda vacío, el build FALLA porque next.config.ts
  está en frontend/, no en la raíz.

VARIABLES DE ENTORNO (Settings → Environment Variables)
-------------------------------------------------------
  NEXT_PUBLIC_SUPABASE_URL       = https://fruijbbatbznxgvathbk.supabase.co
  NEXT_PUBLIC_SUPABASE_ANON_KEY  = (tu anon key de Supabase)
  NEXT_PUBLIC_BACKEND_URL        = (URL pública del backend FastAPI, si existe)
  STOP_THRESHOLD                 = 95   (opcional, parada temprana CAT)

  Nota: Sin backend FastAPI, la app usa fallback JS local automáticamente.
  Nota: Sin Supabase, las API routes usan datos mock de /mocks.

DESPLIEGUE POR CLI (cuando npm funcione)
-----------------------------------------
  npm install -g vercel
  cd frontend
  vercel login
  vercel --prod

  O desde la raíz con root directory:
  vercel --prod --cwd frontend

DESPLIEGUE POR GIT (recomendado)
--------------------------------
  1. Push de la rama a GitHub
  2. Conectar repo en Vercel (Root Directory = frontend)
  3. Cada push a main/master dispara deploy automático

VERIFICACIÓN POST-DEPLOY
------------------------
  [ ] https://tu-proyecto.vercel.app carga la landing
  [ ] /auth muestra QuickLogin (demo)
  [ ] Login Estudiante → /test funciona con mocks
  [ ] Login Admin → /admin/dashboard muestra gráficos
  [ ] Login Psicólogo → /psicologo/estudiantes lista mock
  [ ] npm run build pasa sin errores TypeScript

ARCHIVOS DE CONFIGURACIÓN
-------------------------
  package.json (raíz)    Delega install/build a frontend/ (postinstall)
  vercel.json (raíz)     outputDirectory = frontend/.next
  frontend/vercel.json   Config alternativa si Root Directory = frontend

  RECOMENDADO en Vercel Dashboard → Settings → General:
    Root Directory = frontend
  Si usas Root Directory = frontend, el package.json de la raíz no es necesario
  para el deploy, pero se mantiene por compatibilidad con build desde la raíz.

MOCKS EN PRODUCCIÓN
-------------------
  Las API routes tienen fallback integrado. En Vercel sin Supabase
  conectado, el sistema sirve datos de demo desde memoria (mismos
  datos que /mocks/*.json).
